SymPad

v0.3.7

Introduction

Sympad is a simple symbolic calculator / scratch pad. It is a labor of love and grew out of a desire for an easy way to calculate a quick integral while studying some math without having to start a shell every time and import a package or fire up a browser and navigate to a site (technincally that last bit is exactly what happens but the response time is better :) This desire for simplicity led to the single script option "sympad.py" which I could plop down on my desktop and execute when needed.

As said, SymPad is a symbolic calculator using SymPy for the math and MathJax for the display in a web browser. It runs as a private http server on your machine and executes the system default browser pointing to itself on startup. User input is intended to be quick, easy and intuitive and is displayed in symbolic form as it is being entered. Sympad will accept LaTeX math formatting as well as Python expressions (or a mix) and evaluate the result symbolically or numerically. The best way to see what it can do is to try a few things...

Quick Start

Try entering any of the following into SymPad:

sin (3\pi / 2)
cos**-1 x
\log_2{8}
\lim_{x \to \infty} 1/x
Limit (\frac1x, x, 0, dir='-')
\sum_{n=0}^oo x^n / n!
\sum_{n=1}**10 Sum (\sum_{l=1}^m l, (m, 1, n))
d/dx x**2
Derivative (\int dx, x)
d**6 / dxdy**2dz**3 x^3 y^3 z^3
Integral (e^{-x^2}, (x, 0, \infty))
\int_0^1 \int_0^x \int_0^y 1 dz dy dx
\int_0^\infty e^{-st} dt
{{1,2},{3,4}}**-1
det({{sin x, -cos x}, {cos x, sin x}})
\begin{matrix} A & B \\ C & D \end{matrix} * {x, y}
{{1,2,3},{4,5,6}}.transpose ()
expand {x+1}**2
factor (x^3 + 3x^2 + 3x + 1)
series (e^x, x, 1, 9)
a, b, c, d = {{1,2},{3,4}}

Usage

You enter expresstions and they get evaluated. The expressions may be in normal Pythonic style like "a * (b + sin (x)**2 + 3/4) / 2", LaTeX such as "a\frac{b+\sin^2{x}+\frac34}{2}" or a mix "a * (b + \sin**x{2} + \frac34) / 2". The input is displayed symbolically as you type. Input history is supported with the up and down arrows.

The symbolic expressions can be copied to the clipboard in various formats. Single-click for a simple native format meant to be pasted back into the input field. A double click-copies the expression in Python format suitable for pasting into a Python shell or source file. Finally, a triple-click will copy the expression in LaTeX format. The single-click native format will always be pasteable back into SymPad whereas the Python and LaTeX formats may or may not be depending on what elements are present.

Types

Numbers

Numbers take the standard integer or floating point form or exponential form such as 123, -2.567, 1e+100, 3E-45 or -1.521e22. The precision for all SymPy Floats used in evaluation is set to the highest precision number present in the equation, so if you ask for the cosine of a number with 50 decimal digits your answer will have at least 50 decimal digits.

Keep in mind that "e" or "E" is the Euler"s number constant $e$ and if you are trying to enter 2 times $e$ plus 22 then do not write it all together as "2e+22" as this will be interpreted to be "2 * 10^22". Instead, use spaces and/or explicit multiplication: "2 * e + 22". Imaginary numbers are entered using the imaginary unit "i" or "I" depending on preference, no Pythonic "j" option at the moment but it can be hacked by setting "j = i" in SymPad.

Variables

Variable names mostly follow LaTeX convention, they are single Latin letters "x", "y", "z", "A", "B", etc... or single Greek letters preceded by a slash such as "\alpha" ($\alpha$), "\epsilon" ($\epsilon$) or "\Psi" ($\Psi$). The variable names "i", "e" and "\pi" represent their respective mathematical constants $i$, $e$ and $\pi$. The whole top-level namespace of the SymPy package is made available as functions or variables. This means that "pi" and "oo" are also available for $\pi$ and $\infty$, as well as any other variables present at the top level. Python's "None", "True" and "False" are also present.

By default lower case "e" and "i" are used to represent Euler's number and the imaginary unit instead of the default SymPy upper case "E" and "I". This is objectively prettier, but can be changed via the "$sympyEI (True)" and "$sympyEI (False)" function. The SymPy constant usage can also be activated via the command line switch "--sympyEI".

Variable names may be followed by various primes ' such as " a' " ($a'$) or " \omega'' " ($\omega''$). Variables may be subscripted with other variables or numbers "x_1" ($x_1$), "y_z" ($y_z$), "\alpha_\omega" ($\alpha_\omega$). This can be extended to silly levels " \gamma_{x_{y_0'}''}''' " ($\gamma_{x_{y_0'}''}'''$).

Differentials are entered as "dx", "\partialx" or "\partial x" and are treated as a single variable. If you want to enter "d" * "x" multiplied implicitly then put a space between them or two spaces between the "\partial" and the "x".

Variables may be assigned values, references to other variables or even entire expressions which will subsequently be substituted for those variables in any future expression evaluation.

Vectors and Matrices

These are specified using curly braces with commas. Vectors are passed as a single level of curlys such as "{1, 2}" or "{x, y, z}". Matrices are passed as nested rows of curlys. A 2x3 matrix would be specified as "{{1, 2, 3}, {4, 5, 6}}", a 1x3 would be "{{1, 2, 3},}" and a 3x1 would be "{{1,},{2,},{3,}}". Note the trailing commas which are needed for the same reason as in Python for tuples of one element (otherwise the curlys would be treated as parenteses instead of vectors / matrices).

Currently I haven't figured out the best way to interface with the SymPy vector module so SymPad coerces vectors to single column matrices. This at least allows computation with them until I figure out how best to use the vector module.

Strings

These exist for the sole purpose of passing string arguments to SymPy functions. They work as expected being enclosed by single or double quotes and supporting escape sequences. For example "Limit (1/x, x, 0, '-')".

Lists

Standard Python bracket enclosed potentially nested lists which like strings exist for the purpose of passing parameters to functions like "Matrix ([[1,2],[3,4]])"

Operations

Addition and Multiplication

Addition is addition and subtraction is subtraction: "a + b", "a - b". Multiplication is explicit with a "*" operator or implicit simply by writing two symbols next to each other so that "a * b" is the same as "ab". There is however a difference between the two in that the implicit version has a higher precedence than the explicit, which means that explicit multiplication will end a limit, sum, derivative or division "/" expression whereas implicit multiplication will not, e.g. "1/xy" = $\frac{1}{xy}$ whereas "1/x*y" = $\frac{1}{x} \cdot y$.

Division also has two operators, the normal "/" which has a fairly low precedence and the LaTeX "\frac" version which has a very high precedence, even higher than exponentiation. So high in fact that parentheses are not needed if using "\frac" as an exponent as in "x^\frac{1}{2}" = $x^\frac{1}{2}$. The "\frac" operation also does not need parentheses if using single digit operands or single letter variables (Latin or Greek) such as "\frac12" = $\frac12$, "\frac\alpha\beta" = $\frac\alpha\beta$ or "\fracxy" = $\frac xy$ (although this last version without a space before the x is not legal in LaTeX but convenient for quick typing here).

Exponentiation

There are two power opearators "^" and "**". They have the same precedence and can be used interchangeably but follow slightly different parsing rules. The "^" operator follows LaTeX rules which only allow a single positive digit or letter variable (Lating or Greek) without the use of curly braces whereas the "**" follows Python rules which allow negative values or variables or functions. To illustrate the diffference: "x**-2" = $x^{-2}$ whereas "x^-2" = $x^-2$ (which makes no sense). Also, "e**log(x)" will work as expected $e^{\log(x)}$ whereas "e^log(x)" = $e^log(x)$.

Logarithms

The natural logarithm of x is specified by "lnx", "\ln x", "log x", "\log{x}". A logarithm in a specific base is specified by "\log_b x" = $\log_b x$, "log_{10}(1000)" = $\log_{10} {1000}$ = 3, etc...

Roots

The square root of x ($\sqrt{x}$) may be entered in any of these forms "sqrtx", "\sqrt x", "sqrt (x)", "\sqrt{x}", with or without the slash. The cube (or any other) root is similar, $\sqrt[3]x$ = "sqrt[3]x", "sqrt[3] (x)" or "\sqrt[3] {x}".

Factorial

"4!" = "24", "x!" = "factorial(x)", "(-0.5)!" = "1.77245385090552" and "simplify(x!/x)" = "gamma(x)".

Limits

To take the limit of an expression "z" as variable "x" approaches "y" enter "\lim_{x \to y} (z)" = $\lim_{x\to y} (z)$. This will only give the limit if it exists and is the same when approaching from both directions, unlike SymPy which defaults to approaching from the positive direction. To specify a direction add "^+" or "^-" to the equation as such: "\lim_{x \to 0^+} 1/x" = $\lim_{x\to 0^+} \frac1x$ = $\infty$ and "\lim_{x \to 0^-} 1/x" = $\lim_{x\to 0^-} \frac1x$ = $-\infty$. Addition and explicit multiplication terminate a limit expression. Limits may also be entered using the standard SymPy syntax "Limit (expression, variable, to)", this defaults to limit from positive direction like SymPy, or you may specify a direction "Limit (expression, variable, to, dir='+-')".

Sums

The summation (finite or infinite) of expression "z" as variable "n" ranges from "a" to "b" is written as "\sum_{n=a}^b (z)" = $\sum_{n=a}^b (z)$. Iterated sums work as expected, "\sum_{n=1}^3 \sum_{m=1}^n m" = $\sum_{n=1}^3 \sum_{m=1}^n m$ = 10. Addition and explicit multiplication terminate a sum expression. Sums may also be entered using the standard SymPy syntax "Sum (expression, (variable, from, to))".

Differentiation

The derivative of expression "z" with respect to "x" is entered as "d/dx z" or "\frac{d}{dx} z" = $\frac{d}{dx} z$. The second derivative is "d^2/dx^2 (z)" or "\frac{d^2}{dx^2} (z)" = $\frac{d^2}{dx^2} (z)$. Using "\partial" ($\partial$) is allowed but must be consistent within the expression. Mixed derivatives are entered as "d^2/dxdy (z)" or "\partial^2 / \partial x\partial y (z)" = $\frac{\partial^2}{\partial x\partial y} (z)$. Derivatives may also be entered using the standard SymPy syntax "Derivative (expression, var1, var2, power2, ...)".

Integration

The anti-derivative of expression "z" with respect to x is written as "\int z dx" = $\int z\ dx$. The definite integral from "a" to "b" is "\int_a^b z dx" = $\int_a^b z\ dx$. "\int dx/x" = $\int \frac1x\ dx$. Iterated and improper integrals also work. Integrals may also be entered using the standard SymPy syntax "Integral (expression, (variable, from, to))".

(In)equalities

Are parsed from the standard Python "=, ==, !=, <, <=, >, >=" or LaTeX "\ne, \neq, \lt, \le, \gt, \ge" symbols. Currently only a single comparison is allowed so an expression like "0 <= x <= 2" is not valid. Note that the "=" and "==" operators are equivalent for SymPy and mapped to the same "Eq" object in expressions but the single "=" operator has a higher precedence than the others and is used by SymPad for variable assignment whereas the double "==" only ever implies comparison.

Parentheses

Explicit "( )" or implicit curly "{ }" parentheses allow prioritization of lower precedence operations over higher ones as usual and also delineate an expression as an input to a function. They may be used interchangeably for single expressions, the only difference being that the implicit version is not drawn if it does not need to be. The case where explicit "( )" parentheses are needed ... explicitly ... is when calling functions in general and always when calling functions which take multiple parameters like "max(1,2,3)". The curly braces are used as shorthand for vectors and matrices if commas are present, but that is a different syntactic usage, curlys with no commas are essentially invisible parentheses.

Member Access

You can access member data or functions of an expression just like in Python with the "." operator. If the attribute name following the dot is followed by a parenthesized expression then it will be treated as a function call, otherwise it is a data member. For example, two ways to get the transpose of a matrix are "{{1,2,3},{4,5,6}}.T" and "{{1,2,3},{4,5,6}}.transpose ()".

Variable Assignment

Using the syntax "var = expression" you can assign some value to be substituted for that variable in all expressions. For example, doing "x = pi" and then evaluating "cos x" will give you "-1". Anything can be assigned to any valid variable, valid mathematical expressions, Python objects like strings or lists or even references to other variables. To delete an assignment use the "del var" function, to delete all assignments do "delall" and to see what variables are currently assigned to, use the "vars" function.

Tuple assignment is supported and as in Python the source can be another tuple or a single iterable object. An impractical example of this would be setting " a, b, c = 'str' " which would give you " a = 's' ", " b = 't' " and " c = 'r' ".

There are two distinct types of assignment that can occur and you should be aware of the difference between them. Copy assignment is the standard type of assignment used by default in most computer languages where if you start with "x = 1" and you then enter "y = x" then the value "1" will be copied to the "y" variable. The value of "y" will be independent of whatever else happens to the variable "x" after this. The other kind of assignment is a reference assignment which will map the source variable instead of copying its value to the target. This means that if you have a reference set like "y = x" and the value of "x" changes then the value of "y" will reflect this new value. The reference assignment happens if you try to assign variables which do not exist, so setting "y = x" before "x" has been created will result in a reference. Otherwise you can force a reference by using the "@()" meta-function. Doing "y = @x" will create a reference to "x" itself instead of copying the value.

Functions

Almost all SymPy functions are available directly just by typing their name, the exceptions being single letter functions like "N" or "S". These can be executed using the escape character "$" before the name. To numerically evaluate the value of "sin (2)" type in "$N (sin (2))". Functions may take multiple comma-separated arguments with optional keyword arguments as well. The keyword argument identifier implementation is hacked into the grammar so if a keyword name can not be entered correctly (due to underscores) then try entering the identifier name as an explicit string such as " 'this_identifier_has_too__many___underscores' = value ".

The standard trigonometric and hyperbolic functions and their inverses can be entered as usual, the forward functions with or without a leading slash: "sin", "\coth". The inverses are entered as Pythonic functions without a slash like "atan" or "acscsh" and the LaTeX versions take a slash and and are spelled out "\arctan". The inverses may also be specified using the common mathematical syntax: "\tan^{-1}x" or "cos**-1 x".

This last form of exponentiating a function is extended as an input shortcut to all functions so that typing "ln**2x" is a quick way to enter "(ln(x))**2". Keep in mind that the "-1" exponent in this context is just a -1 and does not specify the inverse function as it does for the forward trigonometric and hyperbolic functions.

Functions don't technically require explicit parentheses in order to allow quick entry like "sqrt2" but for any parameter more complicated than another function or variable to a power they will be needed. Functions which take more than one parameter always require explicit parentheses.

Most functions which have an explicit mathematical display syntax are translated on the fly for correct rendering. These include the functions "abs/Abs (x)" which are translated to the standard bar syntax for absolute value "|x|", the "factorial (x)" function is identical to writing "x!" and "exp (x)" is the same as writing "e^x". Other functions which are translated are "Derivative", "diff", "Integral", "integrate", "Limit", "limit", "Matrix", "pow", "Pow" and "Sum".

The "$" escape character allows you to execute arbitrary functions which are not normally accepted by the grammar. Function names specifically recognized by the grammar are only those specified in the SymPy module and a few builtins. When functions are entered using the "$" character then many more __builtin__ functions may be accessed, for whatever reason, whether useful or not. Try entering "$print ('Hello World...')" and have a look at the server output. Note that only the non-dangerous __builtin__ functions are specifically included in this list, functions like "eval", "exec" and many more have been left out and are not accessible.

Notes

WARNING! This http server implementation is nowhere near secure, this as well as the posibility of execution of arbitrary Python functions means you should never leave this server open to the internet by serving on an IP address visible to the external world.

Due to mixing operators from Python and LaTeX the grammar may be a little wonky in places so if something doesn't seem to work as it should try wrapping it in parentheses or putting a space between the problematic elements.

There is a special use for the "_" underscore character aside from variable subscripting which is the same as in the Python interactive shell in that it represents the last expression successfully evaluated. To see this in action type in "1" and hit Enter, then type in "expand ((x+1)*_)" and hit Enter. Repeat this several times using the up arrow.

If you are getting results which are just plain wrong, check to see if you have any variables mapped which would be changing the evaluation.

There are many SymPy objects which SymPad does not understand natively yet. In any case where such an object is the result of an evalutation then the SymPy LaTeX representation will be used for the displayed answer and the SymPy str version of the object will be used as the Python copy string. This may or may not allow you to paste the Python string back into SymPad to continue working with the result. A single-click copy of the result will have the element(s) which was/were not understood replaced with "nan".
